  <files>app/src/lib/auth.ts</files>
  <action>
Create `app/src/lib/auth.ts` with all auth flow functions. This module bridges Supabase auth with the community system.

Export the following functions:

1. `getCommunityStatus()` — Returns the current community state:
   - `'none'` — user hasn't joined (no auth session)
   - `'anonymous'` — joined but no email
   - `'verified'` — has email but no username
   - `'member'` — has email + username + public profile
   Uses `getAuthUser()` from supabase.ts and queries profiles table.

2. `joinCommunity()` — Calls `supabase.auth.signInAnonymously()`. Returns the user object. On success, the auto-trigger in Supabase creates a profile row. Also reads current XP level from localStorage (`getXP()` from storage.ts) and updates the profile's xp_level and league fields.

3. `upgradeToEmail(email: string)` — Calls `supabase.auth.updateUser({ email })` which sends a magic link. Returns `{ success: boolean; error?: string }`. The email param must be trimmed and validated (basic format check: includes @ and .). Use bilingual error messages via a `lang` parameter or return error keys.

4. `setUsername(username: string)` — Validates username (3-20 chars, `/^[a-zA-Z0-9_]+$/`), checks uniqueness by querying profiles where username = input (case-insensitive via `.ilike('username', username)`), then updates own profile row with username, is_public = true. Returns `{ success: boolean; error?: string }`.

5. `getProfile(userId?: string)` — Fetches profile from Supabase. If no userId, fetches own profile. Returns `Profile | null`.

6. `updateProfileStats(xpLevel: number, streak: number)` — Updates own profile row with current XP level, streak, and computed league. Called by sync engine after syncing workout data.

Import `getXP, getStreak` from storage.ts and `getLeague` from types/community.ts. Import `getSupabase, getAuthUser, isAnonymousUser` from supabase.ts.

Type: `export type CommunityStatus = 'none' | 'anonymous' | 'verified' | 'member';`
