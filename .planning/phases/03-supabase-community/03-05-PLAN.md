Structure:
1. Title: "Activity Feed" / "Actividad"
2. Feed list: FeedItem cards in reverse chronological order
3. Load more button at bottom (paginate 20 at a time)

Data fetching:
- Query `activity_feed` table joined with `profiles` (for username) and `kudos` (for count + user's kudos state).
- Supabase query:
```typescript
const sb = await getSupabase();
const user = await getAuthUser();
const { data } = await sb
  .from('activity_feed')
  .select(`
    *,
    profiles!activity_feed_user_id_fkey(username),
    kudos(id, user_id)
  `)
  .order('created_at', { ascending: false })
  .range(offset, offset + 19);
```
Then map the results to FeedItem type, computing kudos_count and user_has_kudosed from the kudos array.

- If insert fails (duplicate or error): revert the optimistic update.

Access control:
- Check `getCommunityStatus()`. If 'none': show CommunityBanner.
- If authenticated (anonymous, verified, or member): show the feed. Anonymous users can view but cannot give kudos (show tooltip "Join to give kudos" / "Unete para dar kudos").

Loading state: Skeleton cards with `animate-pulse`.
